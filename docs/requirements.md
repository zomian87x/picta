# Picta — 要件定義書

## 1. 概要

**アプリ名**: Picta
**プラットフォーム**: macOS / Windows デスクトップアプリ (Electron)
**目的**: Google Gemini の画像生成・編集 API を利用した画像生成デスクトップアプリケーション
**配布形式**: macOS: `.dmg` / `.app`、Windows: `.exe`（NSIS インストーラ）(electron-builder)

---

## 2. 機能要件

### 2.1 画像生成・編集（コア機能）

| ID | 機能 | 説明 |
|----|------|------|
| F-001 | テキストから画像生成 | プロンプト入力のみで画像を生成（text-to-image） |
| F-002 | 画像編集 | 入力画像＋プロンプトで画像を編集（image-to-image） |
| F-003 | 複数画像入力 | ユーザ設定で指定した枚数のスロットを表示（デフォルト 2 枠＋キャンバス、最大 3 枚を API に送信） |
| F-004 | キャンバス機能 | フリーハンド描画ツールでアノテーション・手書き入力が可能 |
| F-005 | アスペクト比選択 | 1:1, 2:3, 3:2, 4:3, 3:4, 16:9, 9:16 から選択（デフォルト 16:9） |
| F-006 | 生成枚数指定 | 1〜4 枚の範囲で同時生成枚数を指定 |
| F-007 | 解像度選択 | 1K / 2K / 4K から選択（デフォルト 2K） |
| F-008 | 画像スロットラベル | 各入力スロットにラベル付与（プロンプトで参照用） |
| F-009 | 入力画像自動リサイズ | 大きすぎる入力画像を自動でダウンスケール（デフォルト長辺 2048px） |

### 2.2 プロンプト機能

| ID | 機能 | 説明 |
|----|------|------|
| F-010 | プロンプト入力 | テキストエリアでプロンプトを入力（日本語・英語対応） |
| F-011 | プロンプトプリセット | ユーザが独自のプリセットを作成・編集・削除できる。デフォルトプリセットも提供 |
| F-012 | プリセット管理 UI | プリセット名＋プロンプト内容を登録・一覧表示・選択・削除できる画面 |

> **注**: 翻訳機能（日本語→英語自動翻訳）は v1 では実装しない。

### 2.3 ユーザ設定

| ID | 機能 | 説明 |
|----|------|------|
| F-020 | API キー設定 | ユーザ自身の Gemini API キーを設定画面から入力・保存。Electron の `safeStorage` で暗号化保存。管理者のキーは共有しない |
| F-021 | テーマ切り替え | ダークモード / ライトモードの切り替え。OS 設定に追従するオプションも用意 |
| F-022 | AI モデル選択 | 使用する Gemini モデルを選択（nano banana / nanobanana 2 / nano banana pro） |
| F-023 | 添付画像枚数設定 | UI に表示する画像入力スロットの数を設定（2〜5 枠、API 送信は最大 3 枚） |
| F-024 | 通知音設定 | 画像生成完了時の通知音 ON/OFF、音の種類を選択可能 |
| F-025 | デフォルト保存先 | 生成画像のデフォルト保存フォルダを指定 |
| F-026 | 設定の永続化 | 設定は OS 標準のアプリデータディレクトリに保存し、アプリ再起動後も維持。macOS: `~/.picta/`、Windows: `%APPDATA%/picta/` |

### 2.4 画像管理

| ID | 機能 | 説明 |
|----|------|------|
| F-030 | 生成履歴（永続） | 生成した画像とメタデータ（プロンプト、日時、モデル等）を永続的に保存。アプリ再起動後も閲覧可能 |
| F-031 | 履歴サムネイル表示 | 履歴一覧にサムネイル画像を表示 |
| F-032 | 画像保存 | 生成画像をローカルに保存（PNG/JPG）。デフォルト保存先またはダイアログで指定 |
| F-033 | クリップボードコピー | 生成画像をワンクリックでクリップボードにコピー |
| F-034 | クリップボード貼り付け | Cmd+V（macOS）/ Ctrl+V（Windows）で画像をスロットに貼り付け |

### 2.5 操作性

| ID | 機能 | 説明 |
|----|------|------|
| F-040 | キーボードショートカット | Cmd/Ctrl+Enter: 生成実行、Cmd/Ctrl+S: 画像保存、Cmd/Ctrl+V: 画像貼り付け 等（OS に応じて自動切替） |
| F-041 | ドラッグ&ドロップ | アプリウィンドウへの画像ドラッグ&ドロップで入力スロットに自動配置 |
| F-042 | 自動アップデート | 新バージョンの検知と通知。ユーザの承認後にアップデート実行 |

## 3. 非機能要件

| ID | 項目 | 要件 |
|----|------|------|
| NF-001 | セキュリティ | API キーは Electron `safeStorage` で暗号化。レンダラープロセスから直接アクセス不可（preload 経由） |
| NF-002 | パフォーマンス | 入力画像の自動リサイズにより API 応答時間を最適化 |
| NF-003 | オフライン対応 | API 接続不可時にエラーメッセージを表示。設定・履歴閲覧はオフラインでも可能 |
| NF-004 | データ保存先 | ユーザデータは OS 標準のアプリデータディレクトリ配下に保存（macOS: `~/.picta/`、Windows: `%APPDATA%/picta/`） |
| NF-005 | 対応 OS | macOS / Windows（初期リリース）。将来的に Linux 対応可能な構成 |

## 4. 技術スタック

| レイヤ | 技術 |
|--------|------|
| デスクトップフレームワーク | Electron |
| フロントエンド | HTML / CSS / JavaScript (vanilla or React) |

| ビルド・配布 | electron-builder |
| 自動アップデート | electron-updater |

## 5. 画面構成

### 5.1 メイン画面（Generate タブ）
- プロンプト入力エリア
- プロンプトプリセット選択
- 画像入力スロット（設定枚数分）
- アスペクト比・解像度・生成枚数・モデル選択
- 生成ボタン
- 生成結果表示（ギャラリー）
- 生成ログ

### 5.2 キャンバス画面（Canvas タブ）
- フリーハンド描画エディタ
- アスペクト比・解像度適用ボタン

### 5.3 履歴画面（History タブ）
- サムネイル付き生成履歴一覧
- メタデータ表示（プロンプト、日時、モデル）

### 5.4 設定画面（Settings）
- API キー入力
- テーマ選択（ダーク / ライト / システム追従）
- AI モデル選択
- 添付画像枚数
- 通知音設定
- デフォルト保存先
- プロンプトプリセット管理

## 6. モデルエイリアス定義

| エイリアス名 | モデル ID | thinking_level |
|-------------|-----------|----------------|
| nano banana | `gemini-3-pro-image-preview` | なし |
